Requesting software
===================

When a software package is useful for multiple users, or it is impossible to
install it yourself, you can ask the support team to provide a central
installation on the VSC cluster of interest. There are a few things to take
into account when submitting a software installation request:

- Recent software versions are strongly preferred because installing legacy
  software on modern hardware or operating systems often poses problems. Please
  state specifically which version you need or consider most appropriate.
- Before requesting software, check whether it is already available on the cluster
  by :ref:`searching the installed modules <module_system_basics>`. It is
  also worth checking if the desired software is available in
  :ref:`EESSI <using-eessi>`.
- Please provide a clear reference to where the requested software can be found
  and how it should be installed.
- Specify if the software has GPU support and if the installation should
  enable it.
- If not included with the software, please provide a test case that allows to
  checking correct functioning (without requiring too many resources).
- Free software is made available to all users. For licensed software, the
  installation can be restricted to a specific VSC group. The user is always
  responsible for providing source code when it is not freely available.
  Similarly, the user is responsible for obtaining licenses or setting up
  access to a license serveri, if applicable.

In addition to these general guidelines, site-specific rules apply:

        https://docs.hpc.ugent.be/software_installation_requests

   .. tab-item:: VUB

      Information on requesting software installations is provided at

   .. tab-item:: UAntwerpen
      :sync: ua

      If you did not find the software you want to use, please do not hesitate
      to contact the UAntwerp support team at hpc@uantwerpen.be.
